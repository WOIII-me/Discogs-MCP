# Support

## Using DIG

- Connection and self-hosting instructions: [README.md](README.md)
- Browser extension instructions: [extension/README.md](extension/README.md)
- Questions and usage discussion: [GitHub Discussions](https://github.com/WOIII-me/Discogs-MCP/discussions)
- Reproducible bugs: [GitHub Issues](https://github.com/WOIII-me/Discogs-MCP/issues)
- Direct support: [help@woiii.me](mailto:help@woiii.me)

When reporting a bug, include the MCP client, hosted/self-hosted setup, affected tool or route,
relevant Discogs release/master ID, and sanitized error text. Remove access tokens, usernames you
do not want public, collection/wantlist contents, and private account data.

## Security and privacy

Do not use Discussions, Issues, or email attachments containing secrets to report a vulnerability.
Follow [SECURITY.md](SECURITY.md) and use GitHub private vulnerability reporting.

Privacy questions can be sent to [help@woiii.me](mailto:help@woiii.me). The public privacy policy
is at [woiii.me/privacy](https://woiii.me/privacy).

## Service expectations

The hosted server is a best-effort public instance backed by the user's Discogs API rate budget.
There is no uptime SLA. Check the [release history](https://github.com/WOIII-me/Discogs-MCP/releases)
and open issues before filing a duplicate report. Self-hosting remains available under the MIT
license.
